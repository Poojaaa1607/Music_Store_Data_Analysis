# Music_Store_Data_Analysis
SQL_Project

Objective

The main objective of this project is to design and analyze a relational database for an e-commerce (music store) system using SQL. The project focuses on creating a structured database schema and extracting meaningful business insights such as customer behavior, sales performance, and product trends.

Steps
1. Database Creation

Created a database (ecom_db) to manage the system.

2. Table Design

Designed multiple tables:

Customers

Categories

Products

Orders

Order Items

Reviews

3. Applying Constraints

Used SQL constraints to ensure data accuracy:

Primary Key

Foreign Key

NOT NULL

CHECK

DEFAULT

ENUM

4. Establishing Relationships

Created relationships between tables:

Customers → Orders (One-to-Many)

Categories → Products

Orders → Order Items

Products → Reviews

5. Data Handling (if added)

Inserted sample data for testing and analysis.

6. Data Analysis (if added)

Performed SQL queries to analyze:

Total sales revenue

Top customers

Best-selling products

Product ratings.


Insights

Identified top-performing products based on sales quantity.

Found high-value customers contributing maximum revenue.

Analyzed overall revenue trends from order data.

Evaluated product quality using customer reviews and ratings.

Observed how product categories impact sales performance
